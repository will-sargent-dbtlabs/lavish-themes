dbt Brief · Dashboard

Overview

This page renders the same comparison content as the rest of the Lavish library, split across tabs instead of one long scroll. Use dashboard mode for briefs with several genuinely distinct sections the user will jump between rather than read start to finish.

Why this theme exists

Captured from a rendered customer-briefing artifact: orange/purple gradient accents, rounded white cards on a soft gray paper, pill-shaped status badges, and - in this variant - CSS-only tabs. Good for account health dashboards, engagement recaps, and anything that benefits from at-a-glance status color plus section navigation.

Design is the rendering of intent. Or, less grandly: the choices you make until the page stops fighting you.

Notes

  • Nest each tab's radio inside its own label; size the input to cover the label with position: absolute; inset: 0; opacity: 0;.
  • Use body:has(#tab-x:checked) #panel-x to show panels — not a forward ~ sibling selector, which can't reach past the label boundary.
  • Close every CSS comment with */. A stray --> silently comments out everything until the next real */.
  • Re-verify inside an actual Lavish session, not just the raw file — the artifact iframe is sandboxed (no allow-same-origin), so page-level script cannot reach in, but real and automated clicks against the page tree still work correctly.
